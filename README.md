# AWS Cloud Security Monitoring Investigation Lab (CloudTrail+CloudWatch)

## Objective

This lab/project aims to develop foundational skills in cloud security monitoring, audit logging, and event investigation using AWS. In this lab, I configured AWS CloudTrail to capture and monitor management events across Amazon S3, EC2, and IAM services. I generated cloud activity by creating and deleting S3 buckets, launching and managing EC2 instances, and creating and removing IAM users. I then investigated the resulting CloudTrail events to identify user actions, API calls, timestamps, source IP addresses, and affected resources. Additionally, I explored Amazon CloudWatch to understand cloud monitoring capabilities and compared its role with CloudTrail in providing visibility into AWS environments. This project provided hands-on experience with AWS CloudTrail, CloudWatch, cloud auditing, security investigations, and event analysis while demonstrating how cloud security teams monitor and investigate activity within AWS environments.

### Skills Learned

This project provided hands-on experience with the following cloud security, monitoring, and investigation skills:
- AWS CloudTrail deployment and configuration
- Cloud audit logging and management event monitoring
- Security event investigation using CloudTrail Event History
- Analysis of AWS API calls and user activity
- Security monitoring and visibility within AWS environments
- Cloud security auditing and forensic investigation techniques
